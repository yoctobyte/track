Pull Policy

Each peer in the table above carries an editable pull policy. The policy is keyed by subproject (the prefix of record_type, e.g. map3d, netinventory, quicktrack). New peers start with all subprojects enabled except map3d, because map3d roots can hold large videos and dense reconstructions. Open a peer's policy panel to flip a subproject on or off, or to set a non-default behavior for new subprojects this peer might publish later.
